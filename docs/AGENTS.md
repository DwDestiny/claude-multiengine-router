# Docs Folder Guide

## Boundary

This folder holds roadmap, design notes, and release preparation docs.

## Rules

- Keep README user-facing; put longer planning notes here.
- Track Phase 2/3 scope separately from Phase 1 MVP.
- Keep Windows support decisions and true-machine validation gaps in `phase-2-windows.md`.
- Do not document private local setup as if it were a general default.
